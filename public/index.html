Notes

Because of the symantic importance of whitespace in both ink's symantics and it's runtime representation; re-formatting ink code in this way can result in slight changes in behaviour.

While in a beta state, you should check that the code output from ballpoint still works the way you are expecting.

However, the output of Ballpoint should be stable: Running Ballpoint on its own output shouldn't result in any changes.
